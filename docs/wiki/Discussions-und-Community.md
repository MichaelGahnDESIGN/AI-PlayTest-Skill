# Discussions und Community

GitHub Discussions ist der richtige Ort für Fragen, Beispiele und Verbesserungen.

## Empfohlene Discussion-Kategorien

Falls du Kategorien pflegen möchtest:

- **General**: allgemeine Fragen.
- **Ideas**: neue Play-Test-Ideen.
- **Show and tell**: echte Beispiel-Playtests.
- **Q&A**: konkrete Probleme.
- **Security & Privacy**: Sicherheits- und Datenschutzfragen.

## Gute Discussion: Beispiel

Titel:

```text
Beispiel: Live-Playtest für Admin-Editor mit Rückbau
```

Inhalt:

```text
Umgebung:
- Live

Rolle:
- Admin/Redakteur

Getestet:
- Datensatz suchen
- Datensatz bearbeiten
- Speichern
- öffentliche Übernahme prüfen
- Rückbau prüfen

Was war schwierig?
- Rückbau musste gegen separate API-Datenbank laufen.

Was sollte der Skill besser machen?
- Vor CMS-Tests explizit Datenbank-Ziel prüfen.
```

## Gute Bug-Diskussion

Titel:

```text
PLAYTEST-Ordner wurde nicht in .gitignore eingetragen
```

Inhalt:

```text
Agent:
- Claude Code / Codex / andere

Befehl:
- /playtest

Erwartet:
- PLAYTEST/ steht in .gitignore

Tatsächlich:
- Ordner wurde angelegt, aber nicht ignoriert

Projektart:
- Node-App / Flutter / Symfony / ...
```

## Keine sensiblen Daten posten

Bitte nicht in Discussions posten:

- Passwörter,
- Tokens,
- private URLs mit Zugangscodes,
- echte Nutzerprofile,
- Zahlungsdaten,
- unverpixelte Admin-Screenshots mit sensiblen Daten.

## Beispiel-Themen für Discussions

- Gute Play-Test-Checklisten für SaaS.
- Play-Tests für Spiele.
- Live-Test mit Backup und Rückbau.
- Playwright mit canvasbasierten Apps.
- Datenschutzfreundliche Testdaten.
- Welche Modelle funktionieren am besten?
- Wie viel soll ein Agent fragen, bevor er startet?
